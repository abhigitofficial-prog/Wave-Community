🧠 Binary Subnetting Trick

Convert CIDR to binary: 11111111.11111111.11111111.11000000 = /26 (192). Network bits = 26, host bits = 6 → 2^6 = 64 IPs.

📡 OSI Model: 7 Layers (Security Focus)

🔐 Layer-based Attacks

  • L1 Physical: Cable tapping, rogue devices
  • L2 Data Link: ARP spoofing, MAC flooding, VLAN hopping
  • L3 Network: IP spoofing, ICMP redirect, route poisoning
  • L4 Transport: SYN flood, port scanning, session hijacking
  • L5 Session: Session replay, man-in-the-middle
  • L6 Presentation: SSL/TLS downgrade, ransomware (encryption abuse)
  • L7 Application: SQLi, XSS, DNS poisoning, HTTP attacks
✅ Defense-in-depth: firewall at L3/L4, IDS/IPS at L2-L7, application gateway, TLS everywhere.

🛡️ Hardening Recommendations

✔ Disable unused ports/services, enable logging, use SSH not Telnet, implement ACLs, segment VLANs, update firmware, enable BPDU guard on access ports.

✔ Zero trust model: micro-segmentation, least privilege, continuous verification.

⚔️ Common Network Attacks

▪ ARP Spoofing: MITM on local network → use Dynamic ARP Inspection (DAI)

▪ DNS Cache Poisoning: Redirects users to malicious sites → DNSSEC

▪ DDoS: Volumetric (UDP flood), protocol (SYN flood), application (HTTP slowloris)

▪ VLAN Hopping: Double tagging, switch spoofing → disable trunk on user ports

▪ Port Scanning: Reconnaissance tool, block with firewall rate-limiting

# Example: Nmap scan for open ports (defensive awareness)

🛠️ Defensive Tools & Analysis

🔎 Packet Analyzers & IDS/IPS

Wireshark/tcpdump: Capture traffic, filter (e.g., `tcp.port == 443`), detect anomalies.

Snort/Suricata: Rule-based intrusion detection (e.g., alert for SQL injection patterns).

Zeek (Bro): Network analysis framework, logs metadata, detects scans.

Security Onion: Linux distro for NSM (full packet capture, alerting).

🧪 Lab & Hardening Tools

Netdiscover / arp-scan: Network mapping.

OpenVAS / Nessus: Vulnerability scanning.

Hping3: Custom packet crafting (testing firewall rules).

Metasploit: For authorised penetration testing.

🌍 Advanced: IPv6, Wireless & Cloud Security

🔹 IPv6 Subnetting Basics

IPv6 uses 128-bit addresses, standard prefix /64 for LAN. SLAAC (Stateless autoconfig). Security: privacy extensions (temporary addresses), ICMPv6 neighbor discovery spoofing → use RA guard, SEND (Secure Neighbor Discovery).

📡 Wireless Security

WPA3 (SAE) replaces WPA2-PSK. Avoid WEP/WPA-TKIP. Use 802.1X (Enterprise) with RADIUS. Disable WPS, enable management frame protection (PMF).

☁️ Cloud Networking (AWS/Azure)

VPC, security groups (stateful firewall), NACLs (stateless), network ACLs, flow logs, DDoS protection (AWS Shield, Azure DDoS). Zero trust: micro-segmentation, service mesh (Istio).

# Example: restrict SSH access to a jump host using VPC security group
aws ec2 authorize-security-group-ingress --group-id sg-xxx --protocol tcp --port 22 --cidr 203.0.113.0/24
