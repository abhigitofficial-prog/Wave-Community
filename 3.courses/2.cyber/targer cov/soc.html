🌐 Networking Fundamentals & OSI Model

📡 OSI Model (7 Layers)

  • 7. Application – HTTP, SMTP, DNS
  • 6. Presentation – TLS, encryption, formatting
  • 5. Session – session mgmt, NetBIOS
  • 4. Transport – TCP, UDP, segmentation
  • 3. Network – IP, routing, ICMP, ARP
  • 2. Data Link – MAC, switching, Ethernet
  • 1. Physical – cables, signals, bits
L7: Web attacks
L4: SYN flood
L2: ARP spoofing

⚡ TCP/IP Model (4 layers)

  • Application (L5-L7)
  • Transport (TCP/UDP)
  • Internet (IP, ICMP)
  • Network Access (Ethernet, Wi‑Fi)

TCP/IP is the practical implementation used on the internet. OSI is a conceptual framework.

🔹 TCP: reliable, connection-oriented. UDP: faster, no guarantees. Common ports: 80, 443, 22, 53.

🔥 OSI Layer Attacks (Layer-by-Layer Threats)

L7: SQLi, XSS, CSRF, phishing
L6: SSL/TLS downgrade, padding oracle
L5: session hijacking, replay attacks
L4: SYN flood, port scanning, UDP reflection
L3: IP spoofing, ICMP redirect, route poisoning
L2: ARP poisoning, MAC flooding, VLAN hopping
L1: cable tapping, signal jamming
⚔️ Cyber Kill Chain & MITRE ATT&CK Framework

🔪 Lockheed Martin Cyber Kill Chain (7 phases)

  1. Reconnaissance – OSINT, scanning
  2. Weaponization – payload creation (exploit, macro)
  3. Delivery – phishing, USB drop, drive‑by
  4. Exploitation – trigger vulnerability
  5. Installation – backdoor, RAT
  6. Command & Control (C2) – beacon out
  7. Actions on Objective – data exfiltration, lateral movement

Defenses can be applied at each phase (e.g., email gateway blocks delivery, EDR blocks installation).

🎯 MITRE ATT&CK Framework

Knowledge base of adversary tactics & techniques based on real-world observations. 14 tactics (e.g., Initial Access, Persistence, Privilege Escalation, Defense Evasion, Credential Access, Discovery, Lateral Movement, Exfiltration). Each tactic contains hundreds of techniques (e.g., T1566 Phishing, T1059 Command & Scripting).

Reconnaissance Resource Development Initial Access Execution Persistence Privilege Escalation Defense Evasion Credential Access Discovery Lateral Movement Collection C2 Exfiltration Impact

Used by red teams for emulation and blue teams for detection gap analysis.

📊 SIEM (Security Information & Event Management)

🔹 Core Functionality

  • Log aggregation – collect data from firewalls, servers, endpoints, applications.
  • Normalization & Parsing – transform diverse logs into common schema.
  • Correlation & Alerting – real‑time rule‑based detection (e.g., multiple failed logins → brute force).
  • Dashboards & Reporting – visualizations, compliance reports (PCI‑DSS, HIPAA).
  • Incident investigation & threat hunting – search across terabytes of log data (SPL/KQL).

🔹 Usage & Deployment

SIEMs (Splunk, QRadar, Sentinel, LogRhythm, Elastic Security) are central to SOC operations. Use cases:

  • Real‑time alert on ransomware indicators
  • User & entity behavior analytics (UEBA)
  • Correlating IDS alerts with VPN logins
  • Historical forensic analysis
📌 Example correlation: 10 failed logins from same source in 5 minutes → trigger "Brute Force" alert, enrich with geo‑IP, notify SOC.

⚙️ SIEM Architecture Components

Agents / forwarders (Sysmon, Winlogbeat) → Syslog or API → SIEM collector → indexer → search head → dashboards. Modern SIEM integrates threat intelligence feeds and SOAR playbooks.

🔥 Firewalls: Types & Log Analysis

🛡️ Firewall Types

  • Packet filtering (stateless) – inspects headers (IP, port). Basic ACLs.
  • Stateful inspection – tracks connection state (e.g., established sessions).
  • Proxy firewall (gateway) – acts as intermediary, inspects application layer.
  • Next‑Gen Firewall (NGFW) – includes IPS, application awareness, deep packet inspection.
  • Web Application Firewall (WAF) – protects web apps (SQLi, XSS).

📝 Firewall Logs & Analysis

Typical log fields: timestamp, action (allow/deny), source/destination IP + port, protocol, bytes, rule ID. Example:

2025-01-15T10:23:45 action=deny src=203.0.113.5 dst=192.168.1.100 proto=TCP dpt=445 rule=1001 "SMB access blocked"

Use SIEM to aggregate firewall logs; look for port scans, brute force, denied connections, policy violations.

📌 Common logs: ASA, pfSense, Palo Alto, Fortinet
📧 Email Security & Secure Email Gateway (SEG)

🛡️ Core Email Protection Layers

  • SPF (Sender Policy Framework) – authorizes sending IPs.
  • DKIM (DomainKeys Identified Mail) – cryptographic signature.
  • DMARC (Domain‑based Message Authentication) – policy for reject/quarantine.
  • Email Gateway (SEG) – filters spam, malware, phishing, URL rewrites, sandboxing.
  • Anti‑phishing & impersonation detection – machine learning on header anomalies.
  • Encryption (TLS/ PGP) – protects in‑transit confidentiality.

🔎 Gateway Functionality

SEG sits between email senders and internal mail servers. It scans attachments (AV, sandbox), rewrites malicious links (time‑of‑click analysis), enforces DMARC, and quarantines suspicious emails. Major SEG vendors: Proofpoint, Mimecast, Cisco Email Security, Microsoft Defender for O365.

📌 Best practice: Enable DMARC reporting, block file types .exe/.scr, disable legacy protocols.

⚠️ Common Email Threats

Business Email Compromise (BEC), spear‑phishing, credential harvesting, ransomware via macro attachments, QR code phishing. Gateway + user training + MFA are essential.
