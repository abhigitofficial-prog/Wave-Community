📖 Core Concepts

Cryptography is the science of secure communication, providing confidentiality, integrity, authentication, and non‑repudiation. In cyber security, it underpins SSL/TLS, disk encryption, digital signatures, password storage, and more.

Confidentiality

Encryption prevents unauthorized reading – symmetric (AES) or asymmetric (RSA).

Integrity

Hash functions and MACs detect tampering (SHA‑256, HMAC).

Authentication

Digital signatures and challenge‑response verify identity.

Non‑repudiation

Signatures provide proof of origin.
📐 Mathematical Foundations (Full)

Modular Arithmetic, Groups, Fields

All modern crypto operates in finite structures. \( \mathbb{Z}_n \) = integers modulo n. The multiplicative group \( \mathbb{Z}_n^* \) has order \( \varphi(n) \) (Euler’s totient). For prime \( p \), \( \mathbb{F}_p \) is a finite field.

\[ \text{Euler's theorem: } a^{\varphi(n)} \equiv 1 \pmod{n} \quad \text{for } \gcd(a,n)=1 \]

The discrete logarithm problem (DLP) in \( \mathbb{F}_p^* \) is: given \( g, h = g^x \), find \( x \). Hard for large primes → basis of DH, DSA, ElGamal.

Prime Numbers, RSA & φ(n)

RSA selects two large primes \( p, q \), modulus \( n = pq \), \( \varphi(n) = (p-1)(q-1) \). Encryption: \( c = m^e \bmod n \), decryption: \( m = c^d \bmod n \) with \( ed \equiv 1 \pmod{\varphi(n)} \). Security relies on integer factorization.

\[ \text{If } \gcd(m,n)=1, \quad (m^e)^d \equiv m \pmod{n} \]

Elliptic Curves (ECC)

Weierstrass form: \( y^2 = x^3 + ax + b \) over \( \mathbb{F}_p \). Group law: point addition \( P + Q \). The ECDLP: find \( k \) such that \( Q = kP \). ECC provides same security as RSA with much smaller keys (256-bit ECC ≈ 3072-bit RSA).

\[ \text{Curve25519: } y^2 = x^3 + 486662x^2 + x \text{ over } \mathbb{F}_{2^{255}-19} \]
🏛️ Classical Ciphers & Cryptanalysis

Caesar / Substitution

Shift cipher (Caesar), frequency analysis breaks it. Vigenère (polyalphabetic) – Kasiski examination.

Enigma & Rotor machines

World War II encryption, broken by Allied cryptanalysis (Bombe). Highlights need for strong modern algorithms.

One‑time pad (OTP)

Information‑theoretically secure if key is truly random, same length as message, never reused. Impractical for large volumes.
🔑 Symmetric Key Cryptography (Deep Dive)

Block Ciphers: AES (Rijndael)

AES-128/192/256, 128-bit block. Rounds: 10, 12, 14. Operations: SubBytes (S‑box), ShiftRows, MixColumns, AddRoundKey. Resistant to linear and differential cryptanalysis.

ChaCha20 (stream cipher, ARX design) – used in TLS 1.3, faster than AES in software.

Stream Ciphers

Generate keystream \( k_i = E_K(IV \| i) \) and XOR plaintext. RC4 (broken), Salsa20, ChaCha20. Requirements: unique IV per key, never reuse key/IV pair.

🔄 Block Cipher Modes of Operation

ECB (Electronic Codebook)

\( C_i = E_K(P_i) \). Patterns leak → insecure.

CBC (Cipher Block Chaining)

\( C_i = E_K(P_i \oplus C_{i-1}) \). Requires IV, padding (PKCS#7). Susceptible to padding oracle attacks.

CFB / OFB

Feedback modes; can act as stream cipher.

CTR (Counter)

\( C_i = P_i \oplus E_K(IV + i) \). Parallelizable, random access, no padding. Recommended.

GCM (Galois/Counter)

Authenticated encryption (AEAD): CTR encryption + GHASH authentication. Provides integrity, confidentiality, and authenticity. Used in TLS 1.3.

XTS (XEX-based Tweaked Codebook)

Disk encryption mode (AES-XTS). Each sector has tweak derived from its index.

\[ \text{GCM: } C = \text{CTR}_K(IV, P),\quad T = \text{GHASH}_H(A, C) \]
🔓 Asymmetric Cryptography (Public Key)

RSA (Rivest‑Shamir‑Adleman)

Key generation: \( n = pq \), \( e \) (65537 common), \( d = e^{-1} \bmod \varphi(n) \). Encryption: \( c = m^e \bmod n \), decryption: \( m = c^d \bmod n \). Padding required: OAEP (PKCS#1 v2.2).

Attacks: low private exponent (Wiener), low plaintext (Coppersmith), side‑channel (timing).

Diffie‑Hellman (DH) & ECDH

Key exchange: parties agree on prime \( p \) and generator \( g \). Each sends \( g^a \bmod p \). Shared secret \( g^{ab} \bmod p \). For elliptic curves: ECDH uses scalar multiplication \( a*Q_B = b*Q_A \).

Elliptic Curve Cryptography (ECC)

Curves: NIST P-256, Curve25519 (X25519 for DH). ECDSA signatures, Ed25519 (EdDSA). Advantages: smaller keys, faster operations.

\[ \text{Ed25519: twisted Edwards curve } -x^2 + y^2 = 1 - \frac{121665}{121666}x^2y^2 \]

ElGamal, DSA

ElGamal encryption (semantically secure with proper randomness). DSA / ECDSA: signature standards.

📇 Hash Functions & Message Authentication

Properties

  • Preimage resistance (one‑way)
  • Second preimage resistance
  • Collision resistance

Algorithms

MD5, SHA-1 (broken). SHA-256, SHA-3 (Keccak), BLAKE2, BLAKE3. SHA-3 uses sponge construction.

HMAC: \( \text{HMAC}(K,m) = H((K \oplus \text{opad}) \| H((K \oplus \text{ipad}) \| m)) \)

⚔️ Cryptographic Attacks (Real‑World)

Brute Force & Birthday

AES-128: \( 2^{128} \) keys – infeasible. Birthday attack finds hash collisions in \( \sqrt{2^n} \). SHA-256 collision resistance 128 bits security.

Meet‑in‑the‑Middle

Weakens 2DES to \( 2^{57} \) complexity. Triple DES remains secure.

Side‑Channel Attacks

Timing (constant‑time code required), power analysis, cache (Prime+Probe). Mitigation: blinding, masking, constant‑time implementations.

Padding Oracle (CBC mode)

POODLE, Lucky13, etc. Fixed by using AEAD (GCM) or encrypt‑then‑MAC.

Quantum Computing Threats

Shor’s algorithm factors integers and solves discrete logs → breaks RSA, ECC, DH. Grover’s algorithm reduces symmetric security by half (AES-128 → \( 2^{64} \) brute force).

Length Extension Attack

Vulnerable to MD‑based hashes (SHA‑1, SHA‑2). Not for SHA‑3 or truncated hashes.

Mathematical Attacks on RSA

Low encryption exponent (\( e=3 \)) with small plaintext → cube root attack. Coppersmith’s theorem, Wiener’s attack for small d. Use strong padding (OAEP).

📜 Public Key Infrastructure (PKI)

X.509 Certificates

Bind identity to public key, signed by Certificate Authority (CA). Fields: subject, issuer, validity, public key, extensions. Chain of trust from root CA.

Revocation: CRL (Certificate Revocation List), OCSP (Online Certificate Status Protocol), OCSP stapling.

Digital Signatures

RSA‑PSS, ECDSA, Ed25519. Signing: \( \sigma = \text{Sign}_{priv}(H(m)) \). Verification ensures integrity and non‑repudiation.

Forward Secrecy – ephemeral keys (ECDHE) protect past sessions even if long‑term key leaked.

🌐 Real‑World Cryptographic Protocols

TLS 1.3 (Transport Layer Security)

Handshake: authenticated key exchange (ECDHE), certificates, AEAD (AES‑GCM, ChaCha20‑Poly1305). Removed weak ciphers (CBC, RC4). 0‑RTT session resumption. Protects confidentiality and integrity of web traffic.

SSH (Secure Shell)

Key exchange (DH or ECDH), user authentication (password, public key), encryption (AES-CTR), MAC. Tunnels remote access.

IPsec (Internet Protocol Security)

ESP (Encapsulating Security Payload) encrypts packets, AH (Authentication Header) provides integrity. IKE for key exchange. Used in VPNs.

🚀 Post‑Quantum Cryptography (NIST standardized)

Lattice‑based

CRYSTALS‑Kyber (key encapsulation), CRYSTALS‑Dilithium (signatures). Security based on Module‑LWE / Module‑SIS problems.

\[ \text{LWE: } (A, b = As + e) \text{ with small error } e \]

Code‑based, Hash‑based

Classic McEliece – Goppa codes (large keys). SPHINCS+ – stateless hash‑based signatures, slow but well‑understood.

Hybrid mode: combine classical (RSA/ECC) + PQC to protect against “harvest now, decrypt later” attacks.

🛡️ Best Practices for Cryptographic Implementation

Use High‑Level Libraries

libsodium, OpenSSL (modern APIs), BoringSSL, AWS‑LC. Never roll your own crypto.

Random Number Generation

Use cryptographically secure PRNG (CSPRNG): /dev/urandom, RtlGenRandom, getrandom(). Entropy seeding is critical.

Key Management

Key rotation, secure storage (HSM, TPM), access control. Never hardcode keys.

Const‑Time & Side‑Channel Resistance

Fix timing leaks (avoid branching on secrets, table lookups). Use constant‑time functions (e.g., libsodium crypto_verify).

Authenticated Encryption Always

Prefer AEAD (ChaCha20‑Poly1305, AES‑GCM) over unauthenticated modes.

Hash Usage

For password storage: use Argon2 (winner of PHC), bcrypt, PBKDF2 with high iteration. Do not store plaintext.
