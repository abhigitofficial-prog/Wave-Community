🎯 Red Teaming: Mission & Mindset

🔴 Red Team

Offensive security specialists simulating real-world adversaries. Goal: test detection, response, and resilience using TTPs (Tactics, Techniques, Procedures) aligned with MITRE ATT&CK.

✅ Assumed breach, stealth, persistence, evasion, reporting actionable findings.

🔵 Blue Team

Defenders: SOC, incident response, threat hunting. Purple team = collaboration to improve defenses.

📌 Red Teaming vs Penetration Testing: Red Team is objective-based (steal data, evade detection), not just finding vulnerabilities.

⚔️ Red Team Attack Lifecycle

🔍 Reconnaissance (OSINT)

Passive/active: DNS enumeration, WHOIS, Shodan, theHarvester, Google dorks. Identify exposed assets and employees.

📌 Weaponization & Delivery

Payload generation (msfvenom), phishing emails, drive-by downloads, USB drops. Evasion techniques: obfuscation, encryption.

🚪 Exploitation

Gain initial foothold: CVE exploits, credential stuffing, social engineering. Tools: Metasploit, Cobalt Strike, Evilginx for MFA bypass.

🕹️ Command & Control (C2)

Establish beacon outbound using HTTPS/DNS, custom C2 frameworks (Covenant, Sliver, Mythic). Domain fronting, CDNs to blend in.

🔄 Lateral Movement & Privilege Escalation

PsExec, WMI, RDP, pass-the-hash, Kerberoasting, Golden Ticket. Dumping credentials: Mimikatz, LSASS memory.

🧹 Persistence & Exfiltration

Scheduled tasks, WMI event subscriptions, registry run keys. Data compression, encrypted channels, staged exfiltration.

🛠️ Core Arsenal (Red Team tools)

🔎 Recon & OSINT

Amass, Recon-ng, Sublist3r, Maltego, Shodan, Censys, theHarvester.

💥 Exploitation

Metasploit, Cobalt Strike (simulate advanced adversary), SearchSploit, Evil-WinRM, impacket.

🧬 C2 & Post-Exploit

🔑 Credential Access

Mimikatz, Rubeus, hashcat, john the ripper, NetNTLM relaying (Responder, Inveigh).

🌐 Web Application

Burp Suite Professional, ZAP, ffuf, SQLmap, XSStrike.

🧪 Evasion & Bypass

Donut, Shellter, Veil, Hyperion, process injection techniques, AMSI bypass.

🏰 Active Directory & Cloud Red Teaming

🔹 AD Attack Primitives

  • Kerberoasting (service accounts crackable hashes)
  • AS-REP Roasting (no preauth users)
  • Pass-the-Hash / Overpass-the-Hash
  • DCSync (mimikatz for replication rights)
  • Golden Ticket / Silver Ticket via krbtgt hash
  • ACL abuse (AdminSDHolder, WriteProperty)

☁️ Cloud Red Team (AWS/Azure)

❄️ Enumeration: pacu, scoutsuite, cloud_enum. ❄️ Privilege escalation via misconfigured IAM roles, EC2 metadata, Lambda backdoors. ❄️ Azure AD reconnaissance: Stormspotter, ROADtools. Persistence via management console backdoor, app registrations.

📡 OSINT & Social Engineering (Human Intelligence)

🔎 OSINT Techniques

Email harvesting (Hunter.io, theHarvester), GitHub recon, breach data (Dehashed, HaveIBeenPwned API), geolocation, metadata analysis. Tools: SpiderFoot, Sherlock, Photon.

🎣 Phishing & Pretexting

GoPhish, Evilginx2 (MFA bypass), SET (Social-Engineer Toolkit). Clone legitimate login portals, use red team domains for awareness training.

Defensive advice: MFA + user awareness + URL filtering.

📄 Reporting & Professionalism

🔴 Red Team Reporting

Executive summary (risk impact), attack narrative (TTPS), timeline, evidence logs, detection gaps, prioritized remediation. Include MITRE ATT&CK mappings.

Tools: GhostWriter (TheCyberNeo), Dradis, WriteHat.

🧠 Purple Team Exercises

Collaborative sessions: emulate attacks, measure detection (e.g., Sysmon, EDR), tune alerts. Result: improved visibility and response.
